1. JDK vs JRE vs JVM
        JDK - Java Development Kit
        JRE - Java Runtime Environment
        JVM - Java Virtual Machine
 
        The machine won't understand the java code we wrote. It only understands bytecode and we can't write bytecode by ourselves so 
        whenever we compile the java sorce code it will be converting into bytecode.
        JVM creates a virtual execution Environment on top of Operating system, which makes Java as Platform independent. our java code 
        will be depending on some dependancies like inbuilt class or some files. and these files will be present inside JRE.
        JVM is part of JRE, so we can't use JVM without JRE. and both togather make JDK

3. Difference between '==' and .equals()
        == will perform address/reference comparision
        .equals() will perform content comparision.
        
        if we are creating two different objects and try to compare, we will get FALSE response, 
        == check whether two reference are pointing to same object.


4. Difference between Stack and Heap memeory
        Stack : Used to store quick variables, Primitive datatypes, object references, method calls, local variables.
        Heap : Used to store objects, Arrays, Instance variables, anything created with "new".

        Stack : Memory management is automatic, it is cleaned automatically.
        Heap : Garbage collector will clean the Heap memeory.

        Stack : Follows LIPO
        Heap : Priority based retrival

7. Encapsulation
        Encapsulation is used to make sure sensetive data is hidden from the user.
                Declare the variable/attribute private.
                Provide public get and set method to access and update the private variable.

        If we try to access private variable/attributes without using getter and setters, we will get the following error.     
                error: <variable/attribute> has private access in Demo


8. Constructor
        Constructor is a special method.
        Constructor name should be same as class name.
        Constructor wont return anything.
        Constructor runs automatically whenever a new object is created.
        It is used to initialize an object.
        It sets initial values
